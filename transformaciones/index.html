transform-origin

Por defecto la propiedad transform-origin tiene el valor 50% 50%;

Alterando el valor de esta propiedad podemos alterar el punto de origen que va a tomar la transición

Ejemplo: transform-origin: top left;

transform: scale()

shorthand: transform: scale("x", "Y");

Aplicando transform-origin: top left;

transform: skew()

Permite deformar el elemento y al igual que rotate solo acepta valores angulares (deg, grad, rad, turn)

shorthand: transform: skews("x", "Y");

La MDN no recomienda utilizar este shorthand porque ya no es soportado por todos los navegadores. Para utilizar dicha transformación deberiamos utilizar transform: skewsX() y transform: skewsY()

transform: skewY(50deg);

Aplicardos propiedades con transform

Transform ejecutara las propiedades de izquierda a derecha una a la vez

Por ello el orden de las propiedades variara el resultado final

transform: rotate(90deg) translate(100px) scale(.3);

transform: scale(.3) rotate(90deg) translate(100px);

will-change

Nos permite decirle a los navegadores que propiedades van a cambiar antes de que susedan. Se utiliza para optimozar recursos pero no se recomienda usarlo mucho, esta destinado a utilizarlo como último recurso con el fin de tratar los problemas de rendimiento existentes, no debe usarse de forma preventiva.

No debe abusarse de las animasiones y transiciones ya que dificultan la accecibilidad al usuario.

Si debes utilizar las propiedades de la capa layout es recomendable utilizar will-change. Ejemplo: will-change: width, heigth;

Esto hara que el navegador obtimise dichas propiedades
